Що таке HTML?

HyperText Markup Language (мова розмітки гіпертексту) — стандартизована мова розмітки документів у Всесвітній павутині. Це набір правил, які використовуються для структурування (розмітки) текстової інформації, додавання зовнішніх зображень, створення таблиць, списків і т.д.

HTML-документ — це звичайний текстовий документ, який має розширення .html. Документ, розмічений за допомогою HTML, інтерпретується браузером, в результаті чого, користувачі бачать не першопочатковий код з елементами розмітки, а підсумковий результат обротки HTML-документу.

W3C

Популярність мови розмітки HTML і використання її в якості основі мережі Інтернет, привело до необхідності управління стандартами HTML. World Wide Web Consortium, або скорочено W3C - організація яка займається розвитком HTML та його стандартів.

Основним завданням консорціуму є забезпечення відкритості стандартів мережі Інтернет та її інтернаціоналізації. Відповідно до цієї ідеї вся документація по регулюванню принципів работи мережі носить статус “рекомендацій”, та є відкритою. Будь-який розробник має право на реалізацію стандартів, без будь-яких обмежень та відрахувань W3C.

Підхід відкритості та рекомендацій, а не зобов'язань є основною ідеєю розвитку мережі Інтернет. Одначе, відкритість стандартів, має і негативну сторону, так як не зобов'язкує розробників притримуватись даного стандарту, що ускладнює реалізацію сайтів під браузери різних розробників.

Теги

Основою HTML є теги, які створюють різні елементи, які задають зміст та структуру підсумкового документу.

Тег (tag, іменована мітка, дескриптор) — елемент мови розмітки гіпертексту. Контент, який міститься між початковим і кінцевим тегом, відображається та розміщується відповідно до властивостей, вказаних в початковому тегу. Для виділення тегів серед іншого тексту документу, використовуються спеціальні символи - кутові дужки (angle brackets).

HTML-документ складається з дерева HTML-елементів та тексту. КОжен елемент позначається у вихідному документі відкриваючим та закриваючим тегом (за рідким виключенням).

Приклад тегу

Відкриваючий тег показує, де починається елемент, закриваючий — де закінчується. Закриваючий тег створюється шляхом додавання слешу / перед іменем тегу. Між відкриваючим та закриваючим тегами знаходиться вміст тега — контент.

Теги можуть вкладатись один в одного, наприклад. При вкладанні потбріно дотримуватись порядок їх закриття (принцип «матрьошки»).

Зазвичай виконується одноразово

  1. Перевіряємо чи працює git. Відкриваємо git bash.
  2. Вводимо в консоль команду git
  3. Тепер вказуємо наше ім'я та пошту, які можна побачити  при коміті в репозиторій
  4. Перевіримо чи вказані наше ім'я та пошта

Команди, які використовуються у терміналі

  • git add . – додає файли в коміт. Зазвичай використовуєтьчя «.» або «*» для додавання усіх файлів. Щоб додати один файл – вказується його імя та розширення 
  • git commit -m "Повідомлення, що описує комміт" – З допомогою цієї команди ми виконуємо «комміт» змін. Тобто вказуємо що поточний зліпок проекту потрібно зберегти. Можна вказати без аргументу -m, в такому випадку відкриється текстоввй редактор по замовчуванню, в якому потрібно буде описати комміт.  Якщо це вийшло випадково, введіть :q та натисніть Enter
  • git push origin master – «Пушимо» або відправляємо в репозиторій наші зміпки проекту. origin – назва віддаленого репозиторію (зазвичай вибирають origin, але в теорії може бути будь-яке слово) master – назва гілки в яку додаємо зміни (master – головна гілка в репозиторії, може бути будь-якою)
  • git init ./ – ініціює пустий репозиторій в поточній директорії . Так само можна вказати будь-який шлях.
  • git remote add origin URL – Додаємо адресу віддаленого репозиторію, в який будемо пушити зміни origin – назва віддаленого репозиторію (зазвичай вибирають origin, але в теорії може бути будь-яке слово)
  • git pull origin master – команда для отримання змін̆ з репозиторію.
